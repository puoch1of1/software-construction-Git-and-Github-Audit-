## GitHub Pages — Summary

GitHub Pages is a free hosting service that lets you publish a static website directly from a GitHub repository.

### How It Works

1. **Create a repository**
   For a personal site, name it:
   `your-username.github.io`

2. **Add your website files**
   Upload files like `index.html`, CSS, and JavaScript.

3. **Push to GitHub**
   Once pushed, GitHub automatically builds and hosts the site.

### Choosing What to Publish

In your repository:

* Go to **Settings → Pages**
* Select the branch (e.g., `main`)
* Select the folder (root or `/docs`)
* Save

GitHub will publish the selected branch/folder as your live site.

### Accessing Your Site

Your website will be available at:

```
https://your-username.github.io/
```

You can also connect a custom domain if needed.

### Disabling Pages

To turn it off:

* Go to **Settings → Pages**
* Remove the source branch/folder
  Or delete the `gh-pages` branch if you used one.

---

### Core Idea

GitHub Pages turns a repository into a live website.
Push code → GitHub builds it → It’s online.

It’s ideal for:

* Portfolios
* Documentation
* Project demos
* Learning experiments
